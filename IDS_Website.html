// 01 — introduction

What is an IDS?

An Intrusion Detection System monitors a computer for suspicious activity and raises an alert when something dangerous is found.

Firewall
Blocks the door
Stops people from getting in. Does not watch what happens inside.
IDS — our project
Watches and alerts
Detects suspicious activity inside and raises an alarm. Does not block.
IPS
Detects and blocks
Like an IDS but also automatically stops the attack.
SIEM
Collects everything
Gathers logs from many systems and analyzes them together.
🏦
Think of a bank: the locked front door = firewall (blocks people from entering). The CCTV camera + security guard = IDS (watches everything happening inside and shouts when something is wrong). The guard who physically stops someone = IPS. Our project builds the CCTV + guard — it watches log files and fires alerts.
// 02 — mechanism

How the IDS works

Five steps happen every time a new line appears in the log file.

1
Log file is written by the OS
Every login, command, and connection is automatically recorded by Linux into /var/log/auth.log. The IDS watches this file.
tail -f /var/log/auth.log
2
New line detected by monitor()
The IDS reads the file continuously. The moment a new line appears, it picks it up immediately — just like tail -f in the terminal.
line = f.readline()
3
Pattern matched by regex
Every line is checked against all 6 rules using regular expressions. If a rule pattern is found in the line, the hit counter increases.
re.search(rule["pattern"], line, re.IGNORECASE)
4
Threshold checked
Not every single match fires an alert. The IDS waits until the count reaches the threshold. SSH Brute Force needs 3 hits — one typo won't trigger it.
if counts[rule["name"]] >= rule["threshold"]:
5
Alert fired and saved
A colored alert prints to the terminal with timestamp, severity, rule name, and the exact log line. It is also written to output/alerts.log permanently.
[ALERT] 2026-03-15 10:01:11 | HIGH | SSH Brute Force
// 03 — detection rules

Attack types detected

Six rules are defined in rules.json. Each has a pattern, threshold, and severity level.

CRITICAL
SQL Injection
Attacker puts SQL code in a URL to steal or destroy the database. Even one attempt fires an alert.
UNION|SELECT|DROP TABLE|'--
Threshold: 1 hit — immediate alert
HIGH
SSH Brute Force
Automated script guessing passwords on the SSH login. Most common attack on public servers.
Failed password
Threshold: 3 hits — allows for honest typos
HIGH
Unauthorized Sudo
A user tries to run admin commands without permission. Insider threat indicator.
NOT in sudoers
Threshold: 1 hit — immediate alert
MEDIUM
Directory Traversal
Attacker uses ../ in URLs to escape the web root and read system files like /etc/passwd.
(\.\./){2,}
Threshold: 1 hit — immediate alert
MEDIUM
Root Login Attempt
Direct login attempts targeting the root account. Should never happen on a hardened server.
Invalid user root
Threshold: 1 hit — immediate alert
MEDIUM
Port Scan
Rapid connections to many ports — attacker mapping what services are running on the system.
Connection closed.*preauth
Threshold: 5 hits — rapid fire pattern

// ids.py — five functions
1
load_rules(path)
Opens rules/rules.json and converts it into a Python list. Each item in the list is one detection rule with name, pattern, threshold, and severity.
2
analyze_line(line, rules, counts)
Takes one log line and checks it against every rule using regex. If a pattern matches, the hit counter for that rule goes up by one. If the counter hits the threshold, it calls alert().
3
alert(rule, line)
Prints a colored alert to the terminal. RED for CRITICAL, YELLOW for HIGH, CYAN for MEDIUM. Shows timestamp, severity, rule name, and the exact log line that triggered it.
4
save_alert(timestamp, severity, name, line)
Writes every alert permanently to output/alerts.log. This means even after closing the terminal, you can review all alerts that fired during the session.
5
monitor(log_path, rules)
The main loop. Opens the log file, jumps to the end, and reads new lines as they appear — exactly like the tail -f command. Runs forever until Ctrl+C is pressed.

// 06 — glossary

Key concepts

Terms every cybersecurity student must know before working with IDS tools.

IDS
Intrusion Detection System — monitors and alerts on suspicious activity.
HIDS
Host-based IDS — monitors one machine's own log files. Our project type.
NIDS
Network-based IDS — monitors live network traffic across all machines.
Wireshark
Network packet analyzer that captures and inspects live traffic in real time.
PyShark
Python wrapper for TShark — lets Python scripts analyze live network packets.
Signature
A known-bad pattern used to identify a known attack type.
Threshold
Minimum number of pattern hits before an alert fires.
False Positive
An alert fired for a normal, harmless event. Threshold reduces these.
False Negative
A real attack that the IDS missed. More dangerous than false positives.
ARP
Address Resolution Protocol — maps IP to MAC. Spoofing it is a common attack.
Regex
Regular expressions — pattern matching language used to search log lines.
Brute Force
Automated script trying thousands of passwords to break into a system.
